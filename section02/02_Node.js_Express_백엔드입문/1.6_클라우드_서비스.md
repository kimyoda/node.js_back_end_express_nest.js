# 1.6 클라우드 서비스

클라우드 이전 IDC(Internet Data Center)라는 건물에 서버를 설치하여 인터넷 서비스를 진행
개발자들이 IDC에 뛰어가는 일도 있었다

그러나, 클라우드를 이용하면 사용자는 물리적 서버를 클라우드 컴퓨팅을 이용하여 서버의 기능을 하는 서비스를 인터넷으로 제공받게 된다.

- 클라우드 서비스는 수많은 서버의 자원을 탄력적으로 제공한다. 기술적으로 하드웨어의 가상화가 필요하다

- 클라우드 서비스를 제공하는 회사는 4개의 회사가 있다.
  - 아마존의 AWS
  - MS의 애저(AZURE)
  - 구글의 GCP
  - 한국의 NCP

대부분 1분 단위의 과금 기준을 가지고 있다.
유저가 갑자기 몰리는 시간에 서버를 여러 대 더 띄워서 대응, 유저가 없을 때 서버를 축소해 비용을 아끼는 등의 전략이 있다

## 클라우드 컴퓨팅의 유형

- 클라우드 서비스는 크게 3가지 유형이 있다.
- IaaS(Infrastructure as a Service): 서버 인프라 환경 서비스
  - 기존의 서버 환경의 인프라를 클라우드 서비스로 제공하는 것(서버, 스토리지, 네트워킹)
- PaaS(Platform as a Service): 애플리케이션 개발 및 실행 환경을 제공
  - 인프라를 구축할 필요 없이 코드만 작성하면 서버 환경이 구성되는 것. 서버 프로비저닝, 리소스 확장 및 기본 인프라 관리
  - AWS Elastic Beanstalk, Azure App Service, App Engine등이 있다.
- SaaS(Software as a Service): 브라우저 기반으로 소프트웨어를 사용
  - 애플리케이션을 브라우저를 통해 사용하도록 해주는 서비스
  - 구글 독스, 마이크로소프트 오피스 365, 어도비 크리에이티브 클라우드 등이 있다.

## 클라우드 개념

**리전(Region)과 가용영역(Availability Zone)**

- 리전: 클라우드 데이터 센터가 위치한 지역 단위(예: `ap-northeast-2` 도쿄)
- 가용영역: 리전 내에 도긻된 데이터센터 집합. 하나의 AZ에 장애가 발생해도 다른 AZ에서 서비스가 지속될 수 있도록 분산 배포한다.

**오토스케일링(Auto Scaling)**
트래픽 변화에 따라 서버 인스턴스 수를 **자동으로 늘리거나 줄이는** 기능이다. 쇼핑몰에서 이벤트 시간에 트래픽이 10배 증가해도 자동으로 서버가 확장, 이벤트가 끝나면 다시 줄어 비용을 아낄 수 있다.

**CDN(Content Delivery Network)**
캐시 서버를 배치, 사용자에게 **가장 가까운 서버에서 콘텐츠를 제공**하는 기술이다. 이미지, 동영상, 정적 파일등의 로딩속를 단축시킨다.

**컨테이너와 쿠버네티스**

- Docker: 애플리케이션을 컨테이너로 패키징해 어떤 환경에서도 동일하게 실행되도록 한다
- Kubernetes(k8s): 수많은 컨테이너를 자동 배포, 확장, 관리하는 오케스트레이션 도구. 클라우드 환경에서 사실상 표준으로 자리 잡았다

온프레미스, 클라우드
| 항목 | 온프레미스 | 클라우드 |
|---|---|---|
| 초기 비용 | 매우 높음 (서버 구매) | 낮음 (사용한 만큼 지불) |
| 확장성 | 제한적, 시간 소요 | 즉시 확장 가능 |
| 유지보수 | 직접 관리 | 클라우드 제공사가 담당 |
| 보안 제어 | 높은 제어권 | 제공사 정책에 의존 |
| 데이터 위치 | 자체 서버에 저장 | 원격 서버에 저장 |
