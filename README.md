# node.js_back_end_express_nest.js

# Node.js 백엔드 개발자 되기

Node.js와 NestJS를 중심으로 백엔드 개발의 기초부터 인증, OAuth, 실시간 통신까지 단계적으로 학습하기 위한 저장소입니다.

이 저장소는 단순히 문법만 익히는 것이 아니라,  
**백엔드 개발자가 실제로 알아야 할 핵심 개념과 서버 개발 흐름을 이해하는 것**을 목표로 합니다.

---

## 📌 프로젝트 소개

백엔드 개발자는 단순히 API만 만드는 역할이 아니라,  
서버 구조를 설계하고, 인증과 인가를 처리하며, 데이터 흐름을 관리하고,  
실시간 통신이나 외부 서비스 연동까지 다룰 수 있어야 합니다.

이 저장소에서는 다음과 같은 흐름으로 학습을 진행합니다.

- 백엔드 개발자가 알아야 할 기본 지식
- NestJS 시작하기
- Node.js로 백엔드 입문하기
- 회원가입과 인증 구현하기
- OAuth를 사용한 구글 로그인 인증
- WebSocket을 사용한 실시간 채팅 구현

즉, **입문 → 구조 이해 → 인증 → 외부 로그인 → 실시간 기능** 순서로  
백엔드 개발의 핵심 흐름을 정리하고 학습하는 저장소입니다.

---

## 🎯 학습 목표

이 저장소를 통해 다음을 목표로 합니다.

- Node.js 기반 백엔드 개발의 전체 흐름 이해하기
- NestJS의 구조와 동작 방식을 익히기
- API 서버가 어떻게 구성되는지 이해하기
- 회원가입 / 로그인 / 인증 흐름 익히기
- OAuth 기반 소셜 로그인 구조 이해하기
- WebSocket 기반 실시간 통신 구조 이해하기
- 실무에서 자주 쓰이는 백엔드 개념을 정리하기

---

## 📚 공부할 내용

### 1. 백엔드 개발자를 위해 먼저 알아두면 좋은 지식

백엔드 개발을 시작하기 전에 반드시 알아야 할 핵심 개념들을 먼저 정리합니다.

- HTTP / HTTPS
- 요청(Request)과 응답(Response)
- REST API 개념
- JSON 데이터 구조
- 클라이언트와 서버의 역할
- 쿠키 / 세션 / 토큰
- 인증(Authentication)과 인가(Authorization)
- 데이터베이스 기초
- 비동기 처리와 이벤트 루프
- 에러 처리와 예외 흐름
- 환경 변수와 설정 관리
- 로그와 디버깅 기초

이 단계에서는 “코드를 어떻게 짜는가”보다  
**서버가 어떤 원리로 동작하는가**를 이해하는 데 집중합니다.

---

### 2. NestJS 시작

NestJS는 Node.js 기반의 서버 프레임워크로,  
규모가 커져도 유지보수하기 좋은 구조를 만들 수 있도록 도와줍니다.

이 파트에서는 다음 내용을 공부합니다.

- NestJS 프로젝트 생성
- 기본 폴더 구조 이해
- Module / Controller / Service 역할
- 의존성 주입(DI)
- DTO와 Validation
- 예외 처리 방식
- Middleware / Guard / Interceptor / Pipe 개념
- 환경 설정 분리

목표는 단순히 NestJS를 실행해보는 것이 아니라,  
**왜 이런 구조로 나뉘는지**를 이해하는 것입니다.

---

### 3. Node.js로 백엔드 입문

Node.js 자체를 기반으로 서버 개발의 기본 감각을 익힙니다.

이 파트에서는 다음을 다룹니다.

- Node.js란 무엇인가
- 런타임과 서버의 개념
- npm / package 관리
- 기본 HTTP 서버 이해
- 라우팅 개념
- 비동기 처리(Promise, async/await)
- 파일 분리와 모듈화
- 서버 개발 시 자주 만나는 문제들

이 단계에서는 NestJS 이전에  
**Node.js 백엔드의 기초 체력**을 다지는 것을 목표로 합니다.

---

### 4. 회원가입과 인증

백엔드 개발에서 매우 중요한 주제인 인증 기능을 학습합니다.

이 파트에서는 다음을 공부합니다.

- 회원가입 흐름 설계
- 로그인 처리
- 비밀번호 암호화
- 토큰 기반 인증
- Access Token / Refresh Token 개념
- 인증이 필요한 API 보호
- 사용자 정보 조회
- 인증 실패 / 예외 처리

이 단계의 핵심은  
“사용자를 어떻게 안전하게 식별하고 보호할 것인가”를 이해하는 것입니다.

---

### 5. OAuth를 사용한 구글 로그인 인증

직접 회원가입/로그인을 만드는 것에서 더 나아가,  
구글 계정을 활용한 소셜 로그인 구조를 학습합니다.

이 파트에서는 다음을 다룹니다.

- OAuth란 무엇인가
- 소셜 로그인이 필요한 이유
- 구글 로그인 동작 흐름
- 인가 코드 기반 로그인 이해
- 사용자 정보 조회 및 회원 연동
- 기존 회원가입 방식과 소셜 로그인 방식의 차이
- 토큰 발급 및 로그인 상태 처리

이 단계에서는 단순 구현보다  
**외부 인증 제공자와 우리 서버가 어떻게 연결되는지**를 이해하는 데 집중합니다.

---

### 6. WebSocket을 사용한 실시간 채팅 구현하기

일반적인 HTTP 요청/응답과 달리,  
실시간 기능은 서버와 클라이언트가 지속적으로 연결되어야 합니다.

이 파트에서는 다음을 공부합니다.

- WebSocket이 필요한 이유
- HTTP와 WebSocket의 차이
- 연결 / 해제 처리
- 메시지 송수신 구조
- 채팅방 개념
- 사용자 입장 / 퇴장 처리
- 실시간 이벤트 흐름
- 확장 시 고려할 점 (인증, 스케일링, 메시지 저장 등)

이 단계의 목표는  
“실시간 서비스가 어떤 구조로 만들어지는가”를 이해하는 것입니다.
